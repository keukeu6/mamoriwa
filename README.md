## 動かし方

`index.html` をブラウザで開くだけで動きます（ビルド不要）。
スマホ縦画面を想定したレイアウトです。PCで見る場合もそのまま中央に表示されます。

※ PWA機能（ホーム画面への追加・オフライン動作）を確認する場合は、`file://` ではなくローカルサーバー経由で開いてください：

```
python3 -m http.server 8931
# → http://localhost:8931 を開く
```

## PWA対応（スマホでアプリとして使う）

Web上に公開（GitHub Pages / Netlify など、HTTPS必須）してスマホでURLを開き、

- **iPhone**: Safariの共有ボタン →「ホーム画面に追加」
- **Android**: Chromeのメニュー →「アプリをインストール」

でホーム画面にアイコンが並び、全画面のアプリとして起動します。

- `manifest.json` — アプリ名・アイコン・テーマ色の定義
- `sw.js` — Service Worker。一度開けばオフラインでも起動可能（ネットワーク優先＋キャッシュフォールバック）
- `icons/` — アプリアイコン（512 / 192 / 180px）

## 構成

- 単一HTMLファイル（HTML / CSS / JavaScript）＋ PWA用ファイル（manifest / sw / icons）
- **記録は端末に保存されます**（localStorage）。閉じても・リロードしても残り、日付が変わると「今日の服薬・履歴」だけ自動リセットされます（体調履歴・予定は保持）
- 画面下の「デモ用：記録をぜんぶリセット」で初期状態に戻せます
- 本人ビューと家族ビューは画面上部のタブで切り替え。同一状態を共有しているため、本人の記録が即座に家族ビューへ反映されます

## 実装済みのMVP 4機能

1. **服薬チェック（中心機能）** — 薬リスト表示、「のんだ」ワンタップ記録＋タイムスタンプ、時刻超過で本人に画面内アラート、30分超過で家族ビューに「未服薬」注意（色・バッジ）
2. **今日のひとこと** — 3択タップ（げんき／ふつう／ふちょう）＋音声入力UI
3. **通院・予定管理** — 予定の登録（家族ビュー側）・表示・3日前からのリマインドバッジ。両ビューで共有
4. **家族ダッシュボード** — 今日のサマリー（服薬・体調・次の予定）、AI要約、記録履歴の時系列表示

## 仕様どおり「簡易表現」にしている箇所

- **音声認識**：録音中の見た目＋固定の文字起こし例で代替
- **AI要約**：LLM APIは未接続。アプリの状態から例文を自動組み立てして表示
- **プッシュ通知**：画面内のアラート・バッジ表現で代替

## みまもりグループ（家族とのリアルタイム同期）

第2工程の先行実装として、**本人のスマホの記録が家族のスマホへリアルタイム反映される**同期機能を組み込んであります。

**つながる仕組み**：家族側が「グループを作る」と6桁のみまもりコードが発行される → 本人側がコードを入力して参加 → 以後、記録はFirebase Firestore（`groups/{コード}`）に保存され、参加端末へリアルタイム配信されます。退出・コード確認は ⚙️ せってい から。

`firebase-config.js` が未設定（`null`）の間は従来どおり端末内保存のみのローカルモードで動きます。

### 同期のセットアップ（チーム1人がやればOK）

1. https://console.firebase.google.com/ でGoogleアカウントでログインし「プロジェクトを追加」（名前は `mamoriwa` など。Googleアナリティクスは無効でよい）
2. 左メニュー「構築 → Firestore Database」→「データベースを作成」→ ロケーションは `asia-northeast1`（東京）→ **本番モード**で作成
3. Firestoreの「ルール」タブを開き、以下に置き換えて「公開」：

   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /groups/{code} {
         allow read, write: if true;
       }
     }
   }
   ```

4. プロジェクトの設定（⚙）→「全般」→「マイアプリ」→ Webアプリ（`</>`）を追加 → 表示される `firebaseConfig` の中身をコピー
5. このリポジトリの `firebase-config.js` に貼り付けて `git push`

### セキュリティ上の注意（プロトタイプの割り切り）

- 上記ルールは「コードを知っていれば誰でも読み書きできる」状態です。6桁コードの推測は可能なので、**実在の個人情報・本名は入れない**こと（デモ・試用の範囲で使う）
- 本実装ではFirebase Authenticationによる認証とルールの厳格化が必要（第2工程の課題）
- Firebase Web設定（apiKey等）は公開されても問題ない仕様です（秘密鍵ではありません）
